## Executive Summary

Oblivion Mesh is a production-grade distributed CDN built to solve the latency cliff in traditional hub-and-spoke delivery architectures. By implementing a **Raft consensus algorithm** over WebRTC data channels, each edge node can autonomously elect a leader and re-route traffic during failure scenarios.

## System Architecture

The system operates on three tiers: **Origin** (immutable source of truth), **Relay** (consensus participants), and **Edge** (client-facing cache nodes). Each tier communicates over encrypted gRPC channels with mutual TLS.

## Implementation Detail

The core routing algorithm uses consistent hashing with virtual nodes to distribute load. When a node fails, the remaining nodes trigger a Raft election within 150ms, ensuring zero noticeable downtime for end users.

Key engineering decisions:
- **WebRTC over gRPC** for browser-native peer connections
- **Redis Streams** for audit log replication
- **Astro** for the monitoring dashboard (zero-JS by default)
